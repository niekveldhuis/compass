In this chapter we will discuss data acquisition and data formatting for four online Assyriological projects: ORACC (2.1), ETCSL, (2.2) CDLI (2.3) and BDTNS (2.4).

The data in CDLI and BDTNS are made available in raw-text format, for instance:

This data format is easy to read for humans (those humans who know Sumerian), but less so for computers. It is necessary to tell the software what data elements belong to the text and what does not (for instance, line numbers and surface indicators). The data in ORACC and [ETCSL][https://etcsl.orinst.ox.ac.uk/] are made available in JSON and XML, respectively. Those formats are very explicit and atomistic. They are not easy to read for humans, but are very flexible for computational usage and allow for multiple levels of annotation at the same time. The data in ORACC and [ETCSL][https://etcsl.orinst.ox.ac.uk/] includes lemmatization, CDLI and BDTNS provide transliteration only.

The Compass project mostly deals with ORACC data, and much of this chapter will provide code and explanations for how to deal with various types of information that is included in the JSON files. The parsing of the [ETCSL](https://etcsl.orinst.ox.ac.uk/ data ) XML files is, to some extent, redundant, because all the ETCSL data have been incorporated into ORACC/epsd2 and can be parsed with the tools for regular ORACC projects. Section 2.3 (BDTNS) demonstrates code for constructing a search engine that ignores sign values - that is, searching for luh will also find sukkal, etc. The code uses both BDTNS data and the ORACC Global Sign List, showing how data from different projects can be mashed into a single tool.

The Chapter 3-6 of Compass will work with ORACC data and will parse that data with the tools demonstrated and explained in section 2.1. Chapter 2 is not needed, however, to follow along in those chapters. The present chapter is primarily meant for researchers who wish to pursue their own computational projects and need a deeper understanding of how the data is acquired and formatted.
